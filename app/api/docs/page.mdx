# Basix Schedule API Documentation

API para integração externa com o sistema de agendamento Basix Schedule.

## Autenticação

Todas as requisições devem incluir um API key no header Authorization:

\`\`\`
Authorization: Bearer bsk_your_api_key_here
\`\`\`

## Endpoints

### GET /api/v1/services

Lista todos os serviços ativos da organização.

**Response:**
\`\`\`json
{
  "success": true,
  "data": [
    {
      "id": "service_id",
      "name": "Corte de Cabelo",
      "description": "Corte masculino ou feminino",
      "duration": 30,
      "price": 50.00
    }
  ]
}
\`\`\`

### GET /api/v1/professionals

Lista todos os profissionais ativos da organização.

**Response:**
\`\`\`json
{
  "success": true,
  "data": [
    {
      "id": "professional_id",
      "name": "João Silva",
      "email": "joao@basix.com",
      "phone": "(11) 98765-4321"
    }
  ]
}
\`\`\`

### GET /api/v1/availability

Consulta horários disponíveis para agendamento.

**Query Parameters:**
- `serviceId` (required): ID do serviço
- `professionalId` (required): ID do profissional
- `date` (required): Data no formato YYYY-MM-DD

**Response:**
\`\`\`json
{
  "success": true,
  "data": {
    "date": "2025-01-28",
    "slots": ["09:00", "09:15", "09:30", "10:00"]
  }
}
\`\`\`

### GET /api/v1/bookings

Lista agendamentos da organização.

**Query Parameters:**
- `status` (optional): Filtrar por status (confirmed, cancelled)
- `startDate` (optional): Data inicial (ISO 8601)
- `endDate` (optional): Data final (ISO 8601)

**Response:**
\`\`\`json
{
  "success": true,
  "data": [
    {
      "id": "booking_id",
      "clientName": "Carlos Oliveira",
      "clientEmail": "carlos@email.com",
      "clientPhone": "(11) 99999-1111",
      "startTime": "2025-01-28T10:00:00Z",
      "endTime": "2025-01-28T10:30:00Z",
      "status": "confirmed",
      "service": {
        "name": "Corte de Cabelo",
        "duration": 30
      },
      "professional": {
        "name": "João Silva"
      }
    }
  ]
}
\`\`\`

### POST /api/v1/bookings

Cria um novo agendamento.

**Request Body:**
\`\`\`json
{
  "serviceId": "service_id",
  "professionalId": "professional_id",
  "clientName": "Carlos Oliveira",
  "clientEmail": "carlos@email.com",
  "clientPhone": "(11) 99999-1111",
  "startTime": "2025-01-28T10:00:00Z",
  "notes": "Cliente preferencial"
}
\`\`\`

**Response:**
\`\`\`json
{
  "success": true,
  "data": {
    "id": "booking_id",
    "clientName": "Carlos Oliveira",
    "startTime": "2025-01-28T10:00:00Z",
    "endTime": "2025-01-28T10:30:00Z",
    "status": "confirmed"
  }
}
\`\`\`

## Códigos de Status

- `200` - Sucesso
- `201` - Criado com sucesso
- `400` - Requisição inválida
- `401` - Não autorizado
- `404` - Não encontrado
- `500` - Erro interno do servidor

## Rate Limiting

As requisições são limitadas a 100 por minuto por API key.
